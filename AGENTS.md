# 仓库工作规则

## Minecraft 服务端最高优先级

进行任何 Minecraft / MC 模组开发、排查、重构、资源或 UI 相关任务时，第一时间先考虑服务器里的问题，并把服务端权威逻辑作为最高优先级。

默认工作顺序：
1. 先判断问题是否来自服务端权威逻辑、服务端状态、事件回调、组件、持久化、指令、地图/数据包加载、权限或同步源头。
2. 再检查客户端显示、HUD、Screen、渲染、按键、文案、资源等表现层。
3. 涉及跨端功能时，必须同时确认服务端真相、同步/包路径、客户端展示，不要只改客户端来掩盖服务端规则问题。
4. 如果服务端和客户端表现冲突，优先以服务端实际运行结果为准，并说明客户端只是表现层或同步结果。
